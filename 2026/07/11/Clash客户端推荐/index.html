Clash 客户端推荐：Clash Verge Rev(macOS) + Clash Meta for Android

2026/07/11 Tools 共 1424 字，约 5 分钟

概述

平台推荐说明
macOSClash Verge RevTauri GUI，内置 mihomo 内核
AndroidClash Meta for Android原 Clash for Android 停更后的继任者

下面分别介绍两款客户端，再讲 Clash 的两种运行模式的区别。

macOS：Clash Verge Rev

仓库：https://github.com/clash-verge-rev/clash-verge-rev

Releases 里下载 .dmg（universal 或对应架构均可），双击拖到 Applications。

选它的理由：

  • 原 Clash for Windows 仓库已 archive，Clash Verge Rev 是目前社区主流的活跃继任者
  • 内置 Clash Meta（mihomo）内核，规则、TUN、订阅、Profile 都支持
  • Tauri（Rust + Web）写的，资源占用低，界面清爽
  • 跨 macOS / Windows / Linux

启动后把订阅 URL 粘进去，启用 TUN 模式即可使用。

Android：Clash Meta for Android

仓库：https://github.com/MetaCubeX/ClashMetaForAndroid

Releases 里的 APK 文件名形如：

cmfa-2.11.31-meta-arm64-v8a-release.apk

字段含义：

段含义
cmfaClash Meta For Android
2.11.31版本号
meta-arm64-v8a内核 + CPU 架构（arm64 8 位地址空间）
release构建类型（区别于 debug/unsigned）
.apk安装包格式

绝大多数 2018 年后的 Android 手机都是 arm64-v8a，无脑下这个就行。装上闪退再考虑 armeabi-v7a（老机器）或 x86_64（模拟器/ChromeOS）。

安装：浏览器下载后点开安装，首次需要在系统设置里授权「安装未知来源应用」。

Clash 的两种运行模式

用户模式（系统代理模式）

Clash 监听在 127.0.0.1:7890，对外提供 HTTP / SOCKS5 代理。系统把「HTTP 代理」「SOCKS 代理」指向这个端口。

app ──HTTP/SOCKS5──> 127.0.0.1:7890 ──> 节点

特点：

  • 只有「会读系统代理」的应用会走代理：浏览器、终端、curl 等
  • 命令行工具要自己 export http_proxy=http://127.0.0.1:7890 https_proxy=...
  • DNS 走系统 DNS，不在 Clash 内解析（防污染能力弱）
  • 不需要任何特殊权限，配置最简单

服务模式（VPN/TUN 模式）

通过系统的 VpnService API（macOS 上是 utun，Windows 上是 wintun）拿到一张虚拟网卡（TUN），所有流量都先到 TUN，再到 Clash 解析 + 分流。

app ──> tun0 (虚拟网卡) ──> Clash ──> 节点

特点：

  • 所有应用、所有协议的流量都会被接管，包括不读系统代理的 app
  • DNS 在 Clash 内解析（FakeIP / 真实 DNS 都行），抗 DNS 污染
  • 抓包、改路由都能做，覆盖最彻底
  • 第一次启动会弹 VpnService 授权框

怎么选

场景推荐
浏览器 + 终端科学上网用户模式够用
App 内访问、游戏、容器/VM 出网必须 TUN
想彻底防 DNS 污染TUN
想让某个 app 走直连绕过代理用户模式更灵活

简记：默认开 TUN，准没错。只有要单独绕过某些 app 的场景才回退到用户模式。
